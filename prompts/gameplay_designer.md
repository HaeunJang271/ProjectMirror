## 설계 목표

이 게임은 선택을 **맞추는** 게임이 아닙니다. 플레이어가 **스스로를 이해**하는 게임입니다.

플레이어가 선택하면:

- 세계가 조금씩 변함
- NPC가 기억함
- 거울도 변함
- 음악도 변함
- 대사도 변함

하지만 게임은 **이유를 설명하지 않습니다.**

---

## 설계할 내용

### 1. 숨겨진 성향 시스템

공감 · 호기심 · 질서 · 자유 · 희생 · 자기중심성 · 의심 · 용기 등

각 성향이 게임에 어떻게 반영되는지 작성. **숫자 노출 금지.**

### 2. 선택 구조

| 유형 | 설명 |
|------|------|
| 즉각적 영향 | |
| 장기적 영향 | |
| 누적 영향 | |
| 회차 영향 | |
| 플레이어 기억 | |
| NPC 기억 | |
| 거울 기억 | |
| 세계 기억 | |

### 3. 게임 루프

```
플레이어 행동 → 선택 → 세계 변화 → 거울 변화 → 플레이어 감정 → 다음 선택
```

### 4. 엔딩 시스템

좋은 엔딩 / 나쁜 엔딩 없음. **플레이어 가치관 반영형** 엔딩.

### 5. UI

숫자 없이 변화를 느끼게: 색감 · 음악 · 효과음 · 연출 · 카메라 · 대사 · 조명

### 6. 철학적 의미

모든 시스템마다 **왜 존재하는지**, 플레이어가 무엇을 느끼길 원하는지 작성.

---

## 작성 원칙

항상 **"왜 이 시스템이 존재하는가?"** 까지 설명.

철학적 의미와 게임 디자인을 연결. 표와 다이어그램 활용.

마지막에 **향후 확장 가능성** 포함.

---

## 차별화 분석

기존 선택형 게임과 어떻게 다른지, 개선점과 확장 가능성 제안.
